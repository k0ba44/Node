🛠 Что делает этот скрипт?
1. Обновление системы

Производит полное обновление системных пакетов (apt update && apt upgrade -y).

2. Среда контейнеризации (Docker)
Устанавливает актуальную официальную версию Docker Engine напрямую из репозитория Docker.

3. Развертывание архитектуры нод
Remnanode: Создает директорию /opt/remnanode и интерактивно открывает редактор nano для вставки вашей конфигурации docker-compose.yml, после чего автоматически запускает контейнеры в фоновом режиме.

Selfsteel (Caddy): Создает структуру директорий в /opt/selfsteel и генерирует оптимальный Caddyfile с поддержкой PROXY protocol, TLS-терминации, кастомных портов и редиректов.

Интерактивный ввод: В процессе выполнения скрипт запросит у вас привязываемый домен для автоматического заполнения файла среды .env (SELF_STEAL_DOMAIN).

Заглушка (Шаблон сайта): В /opt/html создается дефолтная минималистичная HTML-страница «It works.», отдающаяся по умолчанию для маскировки.

4. Комплексная безопасность и Файрвол (UFW)
Автоматически настраивает и принудительно включает ufw, открывая следующие порты:

22/tcp — SSH-доступ.

80/tcp & 443 — Веб-трафик (HTTP/HTTPS).

2222 — Альтернативный порт управления.

9100/tcp — Сбор метрик (Node Exporter). Важно: порт изолирован и открыт исключительно для входящих запросов с доверенного IP-адреса сервера мониторинга 87.121.89.33.

5. Защита от брутфорса (Fail2Ban)
Устанавливает службу fail2ban.

Полностью перезаписывает /etc/fail2ban/jail.local жесткой и оптимизированной конфигурацией.

Конфигурация по умолчанию включает защиту секции [sshd]: при 3 неудачных попытках авторизации в течение 10 минут (findtime), IP-адрес атакующего блокируется на 999 часов (bantime).

6. Мониторинг (Node Exporter / Grafana)
Разворачивает официальный Docker-контейнер prom/node-exporter в режиме host сети с пробросом корневой файловой системы хоста в режиме read-only.

Позволяет централизованно собирать метрики железа сервера (нагрузка на CPU, RAM, диски, сеть) вашей Grafana/Prometheus системой через защищенный порт 9100.

📂 Структура путей на сервере
После работы скрипта на сервере будут развернуты следующие компоненты:

/opt/remnanode/ — Конфигурационные файлы ядра ноды.

/opt/selfsteel/ — Веб-сервер Caddy (Caddyfile, .env, логи веб-сервера).

/opt/html/index.html — Статический контент для маскировки (мапится в контейнер Caddy).

/etc/fail2ban/jail.local — Глобальный файл правил блокировок Fail2Ban.

📊 Полезные команды после установки
Просмотр логов контейнеров в реальном времени:

Логи Remnanode:

Bash
cd /opt/remnanode && docker compose logs -f -t
Логи Selfsteel (Caddy):

Bash
cd /opt/selfsteel && docker compose logs -f -t
Проверить статус заблокированных по SSH IP-адресов:

Bash
fail2ban-client status sshd
Проверить текущие правила файрвола:

Bash
sudo ufw status verbose
